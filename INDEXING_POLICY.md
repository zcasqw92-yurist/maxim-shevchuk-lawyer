# Политика индексации сайта

## Постоянная установка проекта

Сайт и весь создаваемый для него контент должны оставаться закрытыми от индексации до отдельного прямого разрешения владельца проекта.

Это правило распространяется на:

- главную и все существующие страницы;
- новые страницы услуг;
- публикации, статьи, категории и подкатегории;
- страницы, создаваемые через будущую админку;
- технические страницы и страницы-переадресации;
- новые тексты, добавляемые в существующие страницы.

## Техническое исполнение

Перед загрузкой артефакта GitHub Pages выполняется обязательный слой `npm run lock:indexing`, который:

1. устанавливает на каждом HTML-файле `noindex,nofollow,noarchive,nosnippet,noimageindex`;
2. убирает все URL из `sitemap.xml`;
3. удаляет файл ключа IndexNow;
4. отмечает сборку как `indexingLocked: true` в `build-info.json`.

После этого выполняется локальная и публичная проверка блокировки.

## Открытие индексации

Открывать сайт или отдельные страницы для индексации запрещено без отдельного прямого указания владельца проекта. Такое изменение должно выполняться отдельным Pull Request с полным SEO-аудитом, обновлением canonical, sitemap, robots.txt и проверкой постоянного домена.
