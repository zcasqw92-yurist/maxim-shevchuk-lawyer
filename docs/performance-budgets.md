# Бюджеты производительности

Автоматическая проверка `npm run test:performance-budget` защищает сайт от незаметного утяжеления при замене демонстрационных визуалов настоящими документами, фотографиями и видео.

## Текущие пределы

| Ресурс | Максимальный вес |
|---|---:|
| Главная HTML-страница | 260 KB |
| Общий CSS | 180 KB |
| Основной JavaScript | 90 KB |
| JavaScript доказательных блоков | 32 KB |
| Постер первого экрана | 260 KB |
| Сумма критических ресурсов первого экрана | 650 KB |
| Один демонстрационный SVG | 120 KB |
| Все изображения сайта | 2.5 MB |

## Обязательные правила

- видеофайлы не должны загружаться до действия пользователя;
- iframe Яндекс Карт не должен присутствовать в исходном HTML;
- сторонние аналитические скрипты не загружаются до согласия;
- постер первого экрана имеет `fetchpriority="high"`;
- документы и остальные визуалы ниже первого экрана используют `loading="lazy"`;
- у изображений сохраняются явные `width` и `height`, чтобы не возникал CLS.

## Требования к будущему видео

Рекомендуемый комплект:

- `intro.webm` — основной современный формат;
- `intro.mp4` — резервный формат;
- `intro-poster.webp` — постер;
- `intro-captions.vtt` — субтитры.

Видео подключается с `preload="none"` и начинает загрузку только после нажатия. На мобильных устройствах не используется автоматическое воспроизведение со звуком.

## Замена визуалов

При замене файлов желательно сохранить текущие имена и пропорции. Перед публикацией нужно запустить полный `npm run check`; превышение бюджета остановит PR или production-деплой до оптимизации файла.
