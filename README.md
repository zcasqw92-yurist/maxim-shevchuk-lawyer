# Персональный сайт юриста Максима Юрьевича Шевчука

Готовый статический многостраничный сайт без CMS и тяжёлых клиентских фреймворков. Поисковые роботы получают полноценный HTML, изображения оптимизированы в WebP, а страницы собираются из одной дизайн-системы.

## Что включено

- главная страница с персональным позиционированием;
- каталог из шести направлений практики;
- отдельная SEO-страница для каждой услуги;
- страница об образовании и подходе юриста;
- каталог административных округов Москвы;
- 12 подготовленных локальных шаблонов;
- контакты, форма обращения и политика конфиденциальности;
- связанная JSON-LD-разметка Person, Organization/LegalService, Service, WebPage, ProfilePage, ContactPage, ImageObject и BreadcrumbList;
- sitemap.xml с точным lastmod и изображениями, robots.txt, canonical, расширенные Open Graph/Twitter и webmanifest;
- адаптивная навигация, доступные формы и диалоги;
- UTM-атрибуция, события форм/контактных кликов и отложенная загрузка аналитики после согласия;
- условные коды подтверждения Вебмастеров, аналитика и IndexNow;
- проверка внутренних ссылок, отдельный SEO-аудит и визуальный smoke-тест;
- защита production-сборки от публикации с тестовыми контактами, черновой политикой или пустыми локальными страницами.

## Запуск

```bash
npm run build
npm run dev
```

Открыть: `http://127.0.0.1:4173`

Проверки:

```bash
npm run validate
npm run audit:seo
npm run test:visual
```

Готовая к размещению версия находится в папке `dist`.

## Обязательно перед публикацией

Открыть `site.config.mjs` и заполнить:

1. реальный домен в `siteUrl`;
2. телефон, email и ссылки на мессенджеры;
3. endpoint формы обратной связи;
4. сведения оператора в политике конфиденциальности;
5. коды подтверждения Google Search Console и Яндекс Вебмастера;
6. при наличии реального офиса — блок `publicOffice`;
7. после проверки запускать production-сборку с переменной `SITE_PRODUCTION=true`.

Если у практики есть подтверждённый публичный офис, после указания точного адреса разметку Organization можно расширить до LegalService/LocalBusiness. До этого не следует передавать поисковикам вымышленный адрес.

Пока `SITE_PRODUCTION` не равен `true`, файл robots.txt закрывает весь прототип от индексации, формы честно сообщают о демонстрационном режиме.

## Районное SEO

Локальные страницы намеренно имеют `noindex,follow`, пока вместо заглушек не появится самостоятельное содержание. Готовые округа можно открывать постепенно, добавляя их коды в `indexableDistricts`, например `['cao', 'zao']`. Переключатель `districtPagesIndexable: true` открывает сразу все округа и предназначен только для полностью проверенного набора.

Для каждой индексируемой районной страницы нужны:

- реальные сценарии обращений по этому району;
- проверенные сведения о территориальной подсудности и органах;
- полезный локальный FAQ;
- уникальное описание порядка работы;
- внутренняя перелинковка с релевантными услугами.

Сборщик не позволит открыть индексирование страницы, пока в ней сохраняются маркеры чернового шаблона. Не следует публиковать десятки страниц, различающихся только названием района: это создаёт риск признания страниц дорвеями.

Подробный аудит, контентные кластеры и план работ находятся в `SEO_AUDIT_AND_ROADMAP.md`. Пошаговая публикация — в `DEPLOYMENT.md`.

## Фотографии и диплом

В проект включены рабочие изображения Максима Юрьевича, созданные по переданному портрету. Оригиналы портрета и диплома исключены из публичного репозитория; публикуется только подготовленная веб-версия диплома с закрытыми регистрационными сведениями.

## Размещение

Содержимое `dist` можно загрузить в корень домена на обычном Nginx/Apache-хостинге, Netlify, Cloudflare Pages или другом статическом хостинге. Для корректной работы красивых URL сервер должен отдавать `index.html` из соответствующей папки.

Для GitHub Pages настроен workflow `.github/workflows/pages.yml`. Он автоматически подставляет адрес репозитория и базовый путь, проверяет сборку и публикует безопасную `noindex`-версию. При подключении собственного домена переменные `SITE_URL` и `SITE_BASE_PATH` нужно заменить на адрес домена и пустой базовый путь.
