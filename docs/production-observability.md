# Контроль production-сборки и аналитики

## Проверка опубликованной версии

Каждая сборка создаёт файл `/build-info.json` со следующими полями:

- `sha` — полный Git commit SHA;
- `version` — короткая версия для CSS и JavaScript;
- `builtAt` — время сборки;
- `contentLastModified` — дата актуализации материалов;
- `production` — режим сборки.

В HTML дополнительно записываются метатеги:

- `site-build-sha`;
- `site-build-version`;
- `site-build-time`.

После GitHub Pages Deploy workflow запрашивает опубликованный `build-info.json` и главную страницу без использования кэша. Деплой считается прошедшим только тогда, когда опубликованный SHA совпадает с `${{ github.sha }}` и на странице присутствуют ключевые блоки новой архитектуры.

CSS и JavaScript получают параметр `?v=<build-version>`, поэтому браузер и CDN не должны сохранять старые ресурсы после публикации нового commit.

## Подключение аналитики

В GitHub → Settings → Secrets and variables → Actions → Variables используются:

| Переменная | Значение |
|---|---|
| `SITE_ANALYTICS_ENABLED` | `true` после получения идентификатора аналитики |
| `YANDEX_METRICA_ID` | числовой идентификатор счётчика Яндекс Метрики |
| `GOOGLE_ANALYTICS_ID` | идентификатор вида `G-XXXXXXXXXX`, если Google Analytics нужен |

Достаточно указать хотя бы один корректный идентификатор. До согласия посетителя аналитические скрипты и события не запускаются.

## События доказательных блоков

- `video_placeholder_open` — открыта заглушка будущего видео;
- `video_placeholder_close` — заглушка закрыта;
- `video_placeholder_contact` — переход к обращению из видеоблока;
- `document_sample_open` — открыт образец документа;
- `document_sample_close` — образец закрыт;
- `document_sample_contact` — обращение после просмотра документа;
- `case_details_open` — раскрыты подробности дополнительного кейса;
- `education_proof_open` — открыт раздел подтверждения образования;
- `office_map_load` — пользователь загрузил интерактивную карту.

Для событий документов передаётся параметр `sample`, для кейса — `case_title`, для всех событий — `page_path`.

## Токены поисковых систем

После получения токенов добавить Repository Variables:

- `GOOGLE_SITE_VERIFICATION`;
- `YANDEX_SITE_VERIFICATION`.

Новый деплой автоматически добавит соответствующие метатеги. После публикации необходимо подтвердить сайт в кабинетах и передать адрес `/sitemap.xml`.
