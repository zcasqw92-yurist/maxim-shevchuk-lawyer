# Чек-лист индексации Google и Яндекса

## Что выполняется автоматически

- публичная сборка создаётся с `index,follow`;
- формируются уникальные `title`, `description`, canonical и JSON-LD;
- создаются `robots.txt` и `sitemap.xml`;
- в sitemap для каждой индексируемой страницы указывается `lastmod`;
- при наличии секрета `INDEXNOW_KEY` после публикации отправляется список URL в IndexNow;
- на сайте публикуются видимые экспертные блоки с внутренними ссылками, авторством и датой актуализации.

## GitHub Actions variables

В настройках репозитория: **Settings → Secrets and variables → Actions → Variables**.

- `SITE_PRODUCTION` — `true` (необязательно: workflow уже использует `true` по умолчанию);
- `SITE_URL` — канонический адрес сайта без завершающего слеша;
- `SITE_BASE_PATH` — `/maxim-shevchuk-lawyer` для текущего GitHub Pages либо пустое значение для отдельного домена;
- `GOOGLE_SITE_VERIFICATION` — содержимое verification-токена Google без HTML-тега;
- `YANDEX_SITE_VERIFICATION` — содержимое verification-токена Яндекса без HTML-тега.

## GitHub Actions secret

В разделе **Secrets** добавить:

- `INDEXNOW_KEY` — случайная строка из 8–128 латинских букв, цифр или дефисов.

Сборка автоматически разместит файл `<ключ>.txt` в корне сайта и после успешного деплоя отправит URL в IndexNow.

## Google Search Console

1. Добавить ресурс с точным публичным URL сайта.
2. Выбрать подтверждение через HTML-тег.
3. Скопировать только значение `content` в `GOOGLE_SITE_VERIFICATION`.
4. После деплоя подтвердить право на сайт.
5. Добавить `https://<домен>/sitemap.xml`.
6. Через проверку URL запросить переобход главной страницы и основных страниц услуг.

Для обычных юридических страниц Google не предоставляет общий API мгновенной индексации. Отправка sitemap и запрос переобхода ускоряют обнаружение, но не гарантируют срок или позицию.

## Яндекс Вебмастер

1. Добавить сайт и выбрать подтверждение через метатег.
2. Скопировать значение в `YANDEX_SITE_VERIFICATION`.
3. После деплоя подтвердить права.
4. Добавить sitemap.
5. Проверить robots.txt, региональность и отсутствие критических диагностик.
6. Отправить главную и изменённые страницы на переобход.
7. Связать сайт с действующей карточкой Яндекс Бизнес.

## После запуска

- не менять дату материала без реального обновления содержания;
- не создавать массовые страницы с почти одинаковым текстом;
- не публиковать скрытые ключевые слова;
- расширять сайт только уникальными страницами под самостоятельное намерение пользователя;
- проверять запросы, показы, клики и среднюю позицию отдельно по каждой странице;
- оценивать не только трафик, но и целевые обращения и оплаченные услуги.
