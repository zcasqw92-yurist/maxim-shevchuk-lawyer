# Публикация без потери SEO

## 1. Заполнить конфигурацию

В `site.config.mjs` указать реальный HTTPS-домен, телефон, email, обработчик формы и коды подтверждения Google Search Console и Яндекс Вебмастера. Если существует публичный офис, заполнить `publicOffice` и включить `enabled: true`. Адрес одновременно появится на странице контактов и в разметке `LegalService`.

Для аналитики сначала утвердить политику обработки данных, затем заполнить идентификаторы и включить `analytics.enabled`. При `requireConsent: true` встроенная панель не загружает сторонние счётчики до согласия посетителя; её формулировки и финальную политику всё равно следует проверить у профильного специалиста. События форм и контактных ссылок уже подготовлены.

Обновлять `contentLastModified` нужно только после существенного изменения текста, разметки или внутренних ссылок. Не менять дату автоматически при каждой сборке.

## 2. Проверить сборку

```bash
npm run build
npm run validate
npm run audit:seo
npm run test:visual
```

Все критические ошибки должны быть устранены.

## 3. Открыть индексацию

Только после заполнения контактов, политики и формы:

```js
production: true
```

Затем снова выполнить сборку. В готовом `dist/robots.txt` должна появиться ссылка на карту сайта, а на индексируемых страницах — `index,follow`.

## 4. Настроить сервер

Пример для Nginx находится в `deploy/nginx.conf.example`. Он обеспечивает:

- единый HTTPS-хост и 301-редирект с `www`;
- корректные URL с каталогами и страницу 404 со статусом 404;
- сжатие текстовых ресурсов;
- безопасное кэширование HTML и статических файлов;
- базовые защитные заголовки.

После публикации вручную проверить:

```text
https://DOMAIN/
https://DOMAIN/robots.txt
https://DOMAIN/sitemap.xml
https://DOMAIN/nesushchestvuyushchaya-stranitsa
```

Последний URL должен возвращать именно HTTP 404, а не 200.

## 5. Подключить поисковые системы

1. Подтвердить домен в Google Search Console и Яндекс Вебмастере.
2. Загрузить `https://DOMAIN/sitemap.xml` в оба сервиса.
3. В Яндекс Вебмастере указать регион «Москва» после размещения подтверждающих контактных данных.
4. Создать и полностью заполнить карточку в Яндекс Бизнесе с тем же именем, телефоном, сайтом и реальным адресом/зоной обслуживания.
5. После публикации или существенных обновлений использовать переобход URL и IndexNow.

Для IndexNow задать ключ в `site.config.mjs`, пересобрать сайт, опубликовать файл ключа и выполнить:

```bash
npm run submit:indexnow -- --dry-run
npm run submit:indexnow
```

## 6. Контроль после запуска

Через 48–72 часа проверить доступность страниц роботам, выбранные канонические URL и ошибки структурированных данных. Еженедельно отслеживать запросы, показы, клики, CTR, страницы входа и обращения; ежемесячно пересматривать контент-план по реальным данным, а не по предположениям.
